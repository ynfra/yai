Act as a senior operator performing a first-time yai stack setup. Goal: go
from a clean clone to a fully running stack with all secrets rotated and all
services healthy.

User input: $ARGUMENTS

## Prerequisites check

```sh
./yai.sh doctor
```

Abort with a clear error if:
- docker CLI or daemon is missing (FAIL in toolchain)
- `jq` is missing → install: `sudo apt-get install -y jq`

If `docker group` shows FAIL/WARN and docker commands fail with "permission
denied", run all `./yai.sh` commands prefixed with `sudo sg docker -c "..."`:
```sh
sudo sg docker -c "./yai.sh init all"
sudo sg docker -c "./yai.sh start postgres"
```
Or log out and back in to apply group membership, then retry without `sudo`.

If the repo directory is owned by root, `init` and `start` also need `sudo`.
Combine both with `sudo sg docker -c "./yai.sh ..."`.

## Step 1 — Generate secrets into `.env.local` files

`yai.sh` loads `.env` then `.env.local` via `--env-file` for every service.
Real secrets go in `.env.local` (gitignored). Never overwrite `.env`.

For each service below, edit `<service>/.env` and replace placeholder values
with real random values (use `openssl rand -hex 32`):

| Service | Keys to set |
|---------|------------|
| `postgres` | `POSTGRES_PASSWORD` |
| `n8n` | `POSTGRES_PASSWORD`, `ENCRYPTION_KEY`, `RUNNERS_AUTH_TOKEN` |
| `litellm` | `LITELLM_MASTER_KEY` (`sk-` prefix), `LITELLM_SALT_KEY`, `DATABASE_URL` (uses postgres password) |
| `langfuse` | `NEXTAUTH_SECRET`, `SALT`, `ENCRYPTION_KEY` (64-char hex), `POSTGRES_PASSWORD`, `CLICKHOUSE_PASSWORD`, `MINIO_ROOT_USER` (`langfuse`), `MINIO_ROOT_PASSWORD`, `REDIS_AUTH` |
| `windmill` | `POSTGRES_PASSWORD` |
| `browserless` | `BROWSERLESS_TOKEN` |
| `firecrawl` | `POSTGRES_PASSWORD`, `BULL_AUTH_KEY` |

LiteLLM's `DATABASE_URL` must use the same `POSTGRES_PASSWORD` set for
`postgres/.env.local`:
```
DATABASE_URL=postgresql://yai:<pg_password>@host.docker.internal:25432/litellm
```

After updating the `.env` files, update `env.sh`:
- Replace the `YAI_POSTGRES_PASSWORD` default with the generated postgres password
- Replace the `YAI_LITELLM_MASTER_KEY` value with the generated master key

## Step 2 — Init data directories

```sh
./yai.sh init all
```

This creates all bind-mount directories and applies correct ownership for
services that run as non-root UIDs:
- ClickHouse → UID 101
- Grafana → UID 472
- n8n → chmod 777 (OrbStack bind-mount quirk)

The postgres data directory must be empty before first start:
```sh
ls postgres/data/   # must be empty
```

## Step 3 — Start postgres, create litellm DB

```sh
./yai.sh start postgres
docker exec yai-postgres psql -U yai -d yai -c 'CREATE DATABASE litellm;'
```

Wait for postgres to be healthy before continuing.

## Step 4 — Start remaining services

`start` accepts multiple service names or `all`:

```sh
./yai.sh start minio qdrant browserless
./yai.sh start litellm n8n windmill firecrawl
./yai.sh start langfuse
./yai.sh start vmetrics vlogs vtraces vector node-exporter
./yai.sh start grafana traefik
```

## Step 5 — Verify

```sh
./yai.sh doctor
```

Expected final state: all services `UP`, all `.ENV env`, all `DATA OK`.
(minio shows `PARTIAL` — normal, init container exits after seeding buckets.)

## Post-setup notes

- **Langfuse API keys** (`YAI_LANGFUSE_PUBLIC_KEY` / `YAI_LANGFUSE_SECRET_KEY`
  in `env.sh`) are project-level keys generated by Langfuse after first login.
  Log in at `http://localhost:23000`, navigate to
  Settings → API Keys, and paste them into `env.sh`.
- **Grafana token** (`YAI_GRAFANA_TOKEN` in `env.sh`): create a service account
  token in Grafana (Administration → Service accounts) and paste it into `env.sh`.
- **LiteLLM models**: add upstream provider API keys (OpenRouter, OpenAI,
  Anthropic, …) to `litellm/.env.local` and model routes to `litellm/config.yml`.
